Pourquoi utiliser un framework backend ?
Techniquement, on peut creer un serveur web sans framework.
Par exemple avec Node.js:
    </> JavaScript 
    import http from "htpp";

    const server = http.createServer((req, res) => {
        res.end("Hello World");
    });

    server.lsiten(3000);

Mais des que l'app devient plus grande, on a besoin de:
- gerer les routes (/users, /products, ...)
- parser le JSOn 
- gerer l'authentification
- valider les donnees
- gerer les erreurs
- se connecter a une DB
- gerer les middlewares
- securiser l'app 
Un framework fournit deja ces outils.

Au lieu de tout ecrire nous-meme:
    Application
        ↓
    Framework
        ↓
    Serveur HTTP
        ↓
    Système d'exploitation

Exemple avce Express
Sans Framework:
    </> JavaScript
    if (req.url === "/users" && req.method === "GET")
    {
        ...
    }

Avec Express:
    </> JavaScript
    app.get("/users", async(req, res) =. {
        ...
    });
C'est plus simple et plus lisible.
