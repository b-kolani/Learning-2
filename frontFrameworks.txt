Pourquoi utiliser un framework frontend ?
On peut creer un site web uniquement avec:
- HTML 
- CSS 
- JavaScript 
Par exemple:

    </> HTML 
    <button id="btn">Compteur</button>
    <p id="count">0</p>

    </> JavaScript
    let count = 0;
    document.getElementById("btn").addEvemtListener("click", () => {
        count++;
        document.getElementById("count").textContent = count;
    });
Cela fonctionne tres bien pour de petist projets 

Mais quand l'app grandit;
- centaines d'ecrans 
- gestion des utilisateurs 
- tableaux de bord 
- messagerie temps reel 
- jeux multijoueurs 
- applications mobiles 
la gestion du DOM devient compliquee.

Les frameworks apportent:
- composants reutilisables
- gestion de l'etat (state)
- routage 
- communication avec les APIs 
- optimisation du rendu 
- outils de developpement

Frameworks frontend par langage 
Dans la pratique, la plupart des frameworks frontend utilisent 
JavaScript ou TypeScript.
Les plus populaires aujourd'hui

React
La technologie la plus utilisee.
    </> JavaScript
    function Counter() {
        const [count, setCount] = useState(0);

        return (
            <button onClick={() => setCount(count + 1)}>
                {count}
            </button>
        );
    }

Next.js 
Construit au-dessus de React.
Ajoute:
- routage automatique 
- rendu serveur (SSR)
- API routes 
- optimisation SEO 
Tres demande en entreprise

Plus simple a apprendre que React pour beaucoup

    </> vue

    </template>
